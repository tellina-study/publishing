# tasks/ — рабочая кухня каждой задачи

Каждая задача (статья, реферат, инфраструктура — **любая**) ведётся здесь как изолированный,
отслеживаемый юнит. `tasks/` — это **процесс/кухня**; публикуемые продукты (текст статьи, варианты
под каналы) живут в своём месте репы (`pieces/`). Разделение: *процесс vs продукт*.

## Раскладка папки задачи

```
tasks/YYYYMMDD_<slug>/
  brief.md        # ПОСТАНОВКА: цель, скоуп, каналы, ship-bar, DO NOT, ссылка на GitHub issue
  plan.md         # ШАГ-0: детальный план + чеклист именно под эту задачу (идём строго по нему)
  log.md          # ЛОГ ПРОГРЕССА: датированные записи — что сделано / решено / удивило
  result.md       # ИТОГ: вердикт + ссылки на артефакты в репе (pieces/...) и опубликованные URL
  owner-taste.md  # (опц.) ожидания Макса к ЭТОЙ задаче — per-task слой зеркала
  materials/      # промежуточные материалы, выгрузки, данные  (gitignored — кухня)
  data/           # данные/датасеты                            (gitignored — кухня)
  scripts/        # одноразовые скрипты под задачу             (трекаются; полезное → scripts/)
```

Что коммитим: `brief / plan / log / result / owner-taste / scripts` — это durable-запись, она
едет с веткой и PR. Что не коммитим: `materials/`, `data/`, архивы — это локальная кухня
(см. `.gitignore`). Если данные надо сохранить — опиши их в `result.md`, не заливай бинарь.

## Жизненный цикл задачи

1. **Ясность.** Не начинаем, пока задача не понятна до конца: сперва изучить базу знаний
   (RAG → wiki → онтология → прошлые материалы), затем уточняющие вопросы **последовательно**.
2. **Ветка + worktree.** Своя ветка (`piece-<slug>` / `kb-<short>`), свой git worktree. Не в `main`.
3. **brief.md → plan.md.** Постановка, потом ШАГ-0 план под задачу. Заводим GitHub issue.
4. **Работа по плану**, лог прогресса в `log.md` по ходу (не задним числом).
5. **Для публикуемого:** lifecycle BRIEF→DRAFT→FACT-CHECK→CRITIQUE→SHIP, артефакты в `pieces/`.
6. **Итог + close-the-loop.** `result.md`; затем wiki / онтология / RAG / decisions через `librarian`.

Старт задачи = `cp -r templates/task tasks/YYYYMMDD_<slug>` и заполнить.
